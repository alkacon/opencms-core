Nehmen Sie an, sie möchten alle Unterverzeichnisse und enthaltenen Dateien eines Verzeichnisse der Gruppe Guest nur zum Lesen zur Verfügung stellen. Ohne Vererbung von Berechtigungen innerhalb von Verzeichnisstrukturen müssten Sie jetzt an jeder dieser Unterdateien und Unterverzeichnissen die Rechte manuell setzen. Eine sehr aufwändige Arbeit. Stellen Sie sich vor, in Zukunft werden weitere Dateien und Unterverzeichnisse in diesesm Verzeichnis erstellt. Die verantwortlichen Redakteure wissen nichts von diesem Rechtemanagement und vergessen, auch für die neuen Ressourcen die Rechte der Gruppe Guest setzen.

Durch Vererbung der Berechtigungen an einer Ressource können Sie einfach an ein Verzeichnis Berechtigungen für eine Gruppe oder einen Benutzer setzen. Automatisch werden alle Unterdateien und Unterverzeichnisse dieser Gruppe oder diesem Benutzer die gleichen Rechte gewähren.

Diese Vererbung kann in den Kind - Ressourcen (Unterdateien und Unterverzeichnisse) nur dadurch manipuliert werden, dass an die Kind - Ressourcen Zugriffsrechte für denselben Benutzer / dieselbe Gruppe neu definiert werden.

Bitte beachten Sie: Wenn in der Vererbungsreihenfolge einer Gruppe / einem Benutzer ein Zugriffsrecht ausdrücklich verboten wurde, so kann ein ausdrückliches Erlauben desselben Rechtes für denselben Benutzer / dieselbe Gruppe dies nicht wieder aufheben, es sei den das Häkchen "Geerbte Überschreiben" wird gesetzt.
