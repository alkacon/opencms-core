The form-based content editor provides a rich user interface for the content managers. All XML content fields can be edited via intuitive widgets.

 

Main features of the content editor:

 

  • Runs on the client side and provides a highly responsive user experience.
  • Automatically generated from an XML schema, no programming required.
  • Provides a large selection of rich user interface widgets.
  • Can be extended with custom user interface widgets for special use cases.
  • Generates XML files that are validated against the XML schema and stored in the database.

 

Overview on the form-based editor.

 

The form-based editor is the standard editor for XML content. It opens when you click the pencil at a content element in the page editor, or when you edit XML content from the workplace. The editor shows all content fields in an intuitively editable way. Depending on the type of the content field, special editor widgets are used.

Figure shows some content opened in the form-based editor. In the example you see:

  • The structure of the content is reflected in the editor's appearance. The order of the editor fields reflects the order of the schema elements and nested schema elements are displayed as nested editor fields, e.g., Text is nested inside of Paragraph.
  • Content fields are grouped into several tabs.
  • Content values of different types are edited via different widgets - compare for example the editor fields for Title, Text and Image values
  • Optional fields (such as Teaser) are shown in gray if not provided
  • If fields can appear more often than already present, a is shown to add more fields
  • If fields are not mandatory they can be cut, clicking the symbol.
    The editor shows the content in one language. The language can be switched via the language selector in the toolbar.

Редактор структурированного документа реализует расширенный интерфейса контент менеджера для управления содержимым документа. Поля структурированного документа в данном редакторе могут быть отредактированы с помощью интуитивно-понятных виджетов.

 

Основные функциональные возможности:

 

  • Запуск на стороне клиента и высокое быстродействие.
  • Автоматическая генерация полей редактора, не требующая разработки дополнительного интерфейса.
  • Широкий выбор виджетов для редактирования элементов.
  • Может быть расширен пользовательскими виджетами.
  • После завершения редактирования структурированный XML документ автоматически проверяется согласно схеме документа и сохраняется в БД.

 

Общие сведения о редакторе.

 

Редактор структурированного документа является стандартным редактором для XML - документов OpenCms. Он открывается при нажатии на пиктограмму "карандаш" элемента контента или при редактировании XML - документа в рабочем месте OpenCms. Редактор предоставляет для каждого из типов полей документа интуитивно - понятный вариант редактирования. В зависимости от типа контента могут быть использованы специализированные виджеты для редактирования значения поля.

 

Пример страницы редактора приведен на изображении ниже. Основные принципы формирования окна редактора:

  • Внешний вид редактора определяется структурой документа. Именование, типы и порядок следования полей также соответствует структуре XML - документа, описываемой в схеме данного документа.
  • Поля могут быть сгруппированы по нескольким вкладкам.
  • Поля различного типа могут редактироваться  помощью ращличных виджетов, например, текст или изображение.
  • Опциональные поля (такие как "Тизер") отображаются серым цветом в случае, если их значения не указаны.
  • Если может быть указано более одного значения поля, дополнительные значения могут быть добавлены с помощью кнопки .
  • Если значение поля не является обязательным, оно может быть удалено путем нажатия на кнопку .
  • Редактор отображает контент на одном выбранном пользователем языке. Язык может быть изменен с помощь селектора языка в верхней части редактора.
