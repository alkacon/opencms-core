Needed Manual Update Steps

OpenCms version: 6.0.2

Press here to open this list in a new window for printing.

Update of the configuration Files

To enable the new features of OpenCms 6.0.2, several configuration files must be modified. Since you might have done changes to your config files already, the following paragraph describes which nodes in the config files must be changed. The configuration files can be located in the WEB-INF/config/ subfolder of your OpenCms installation.

opencms-system.xml

Exchange the <resourceinit/> node with 

<resourceinit>
  <resourceinithandler class="org.opencms.file.CmsBackupResourceHandler"/>
</resourceinit>

Add the <content-notification> node behind the <resultcache> node: 

<content-notification>
  <notification-time>365</notification-time>
  <notification-project>Offline</notification-project>
</content-notification>

opencms.workplace.xml

You can find all modifications in the file opencms-workplace_changes.html. You can then copy the modifications from there and use them in your own configuration files. Exchange the <explorertypes> node with the one found in the opencms-workplace_changes.html. Those modifications are not listed here because of their size.

Completing the update

Restart the servlet container after you made the changes to the configuration files. Login to OpenCms and switch to the Adminstration view. Select ”Flex Cache Adminstration” and press the ”Purge JSP respository” button there. After this, the update process has been completed.
